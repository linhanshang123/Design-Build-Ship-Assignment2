## Project
Build Assignment 2 as a Day Planner web app.

## Current stage
We are currently in implementation, verification, and style exploration.

## Goal
Create a lightweight multi-page planner that helps a user manage tasks and notes by day.

## Tech
- Next.js
- Tailwind CSS
- App Router
- Playwright MCP

## Implemented routes
- `/` home page
- `/new` add task form
- `/day/[date]` dynamic day page
- `/week` week overview

## Current priorities
1. Make time tracking possible and visible
2. Add more useful tools
3. Verify the `/new` form flow and day/week updates
4. Keep the app simple and stable
5. Prepare for Vercel deployment
6. Keep commit history clean and meaningful

## Constraints
- Use client-side state only
- No database
- No auth
- No external APIs

## Working style
- Work in small steps
- Save stable versions before exploring a new style
- Do not over-engineer
- Prefer polish over adding extra features
- Suggest commits, but do not auto-commit unless asked

## Style exploration focus
Explore new visual directions while keeping the existing route structure and interactions intact.
